# 3xui Telegram Bot (Host/VPS Runtime)

This project now runs **directly on your host/VPS** (no Docker, no compose).

It includes:
- Telegram bot webhook service (`telegram_bot.py`)
- Admin web panel (`admin_web.py`)
- Nginx reverse proxy + TLS (Let's Encrypt)
- Host-side service management scripts (`scripts/*.sh`)

---

## Quick install

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/03hitalin21/3xui-tgbot/New7/install.sh)
```

The installer:
1. installs runtime dependencies (`python3`, `venv`, `nginx`, `curl`, `git`, etc.)
2. clones/updates the repo
3. opens an interactive setup menu

---

## Recommended setup flow (installer menu)

1. **Configure app (.env)**
2. **Acquire/configure TLS certificate (Let's Encrypt)**
3. **Check TLS certificate status**
4. **Start / restart host services**
5. **Set Telegram webhook now**
6. **Run health checks**

Option `7` runs the full chain automatically.

---

## Runtime architecture (host mode)

- `scripts/manage_services.sh` starts/stops local Python processes and stores PID files.
- `scripts/configure_nginx.sh` renders `nginx/nginx.conf` into `/etc/nginx/conf.d/tgbot.conf` via `envsubst` and restarts nginx.
- `scripts/setup_ssl.sh` issues/reuses Let's Encrypt certificates and installs renewal cron.
- `scripts/set_webhook.sh` registers Telegram webhook and validates status.

## Required `.env` values

At minimum:
- `TELEGRAM_BOT_TOKEN`
- `XUI_BASE_URL`
- `XUI_USERNAME`
- `XUI_PASSWORD`
- `XUI_SERVER_HOST`
- `WEBHOOK_BASE_URL`
- `SSL_DOMAIN`
- `LETSENCRYPT_EMAIL`

Also important:
- `WEBHOOK_PATH` (default: `telegram/<random>` from installer)
- `WEBHOOK_PORT` (default: `8443`)
- `ADMIN_WEB_PORT` (default: `8080`)
- `WEBHOOK_SECRET_TOKEN` (recommended)

See `config/bot.env.example` for full list.

## Common setup failure: malformed `.env`

If you see messages like:
- `Ignoring malformed .env line: admin`
- `.env: line X: <token>: command not found`

it means `.env` contains a broken line (usually a value spilled onto a new line).

### Fix
1. Re-run installer option **1** (Configure app).
2. Ensure each value is a **single line** (no pasted line breaks).
3. Save and continue with options **2 -> 4 -> 5 -> 6**.

You can quickly inspect `.env` with:
```bash
nl -ba .env
```

Every non-comment line must be `KEY=value`.

## 502 Bad Gateway troubleshooting (Telegram webhook)

If `getWebhookInfo` shows `last_error_message` with **502 Bad Gateway**, usually nginx is reachable but backend webhook service is not healthy/reachable.

### 1) Check bot/admin process status
```bash
./scripts/manage_services.sh status
```

### 2) Check nginx configuration
```bash
nginx -t
```

### 3) Restart stack in correct order
```bash
./scripts/manage_services.sh restart
./scripts/configure_nginx.sh
./scripts/set_webhook.sh
```

### 4) Verify port alignment
Ensure these match:
- `.env` `WEBHOOK_PORT`
- `telegram_bot.py` runtime port (read from `.env`)
- nginx upstream generated by template (`proxy_pass http://127.0.0.1:${WEBHOOK_PORT}`)

### 5) Validate webhook URL/dns
- `WEBHOOK_BASE_URL` must point to this VPS
- DNS A/AAAA records must resolve correctly
- port `443` must be reachable from internet

### 6) Check logs
```bash
tail -n 100 logs/bot.log
tail -n 100 logs/admin_web.log
journalctl -u nginx -n 100 --no-pager
```

## Useful commands

```bash
# Start services
./scripts/manage_services.sh start

# Stop services
./scripts/manage_services.sh stop

# Restart services
./scripts/manage_services.sh restart

# Service status
./scripts/manage_services.sh status

# Configure nginx from template + restart
./scripts/configure_nginx.sh

# Acquire/check certificate
./scripts/setup_ssl.sh acquire
./scripts/setup_ssl.sh check

# Register webhook
./scripts/set_webhook.sh

# Full health check
./scripts/health_check.sh
```
